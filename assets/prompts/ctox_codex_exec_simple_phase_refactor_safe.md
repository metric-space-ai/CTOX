Main meta-skill: `refactor-safe`

Use inside CTOX:

- Read the refactor boundary and done gate from `Focus`.
- Respect `Anchors` and `Verified evidence` so behavior does not drift.
- If behavior changes or new open work appears, stop and re-classify.

- Run a baseline check first.
- Make one mechanical change at a time.
- Re-run the same check after each step.
- Keep behavior the same.
- If behavior changes, stop.
- Do not mix refactor work with bug fixes or features.
