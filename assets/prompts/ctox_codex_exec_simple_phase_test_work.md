Main meta-skill: `test-work`

Use inside CTOX:

- Read the test target and done gate from `Focus`.
- Keep expected behavior aligned with `Verified evidence` and `Anchors`.
- If the work turns into a product fix, stop and re-classify instead of silently widening scope.

- Change tests only by default.
- Keep one behavior per test.
- Run the smallest exact test first.
- Then run the nearest related tests.
- If the test exposes a product bug, stop and re-classify to `bug-fix`.
- Do not silently change production behavior in this phase.
