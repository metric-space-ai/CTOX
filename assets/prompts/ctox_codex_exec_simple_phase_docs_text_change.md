Main meta-skill: `docs-text-change`

Use inside CTOX:

- Read the wording task from `Latest user turn` and `Focus`.
- Keep documentation and prompt text aligned with `Verified evidence`, `Anchors`, and real behavior.
- This phase changes text only unless the task clearly expands.

- Use when the task is mainly docs, prose, prompts, markdown, comments, or text updates.
- Change one message or claim at a time.
- Keep wording simple and aligned with real behavior.
- Verify nearby wording, references, and examples after each step.
- If the task turns into a real code or ops change, stop and re-classify.
