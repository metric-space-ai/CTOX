Main meta-skill: `code-review`

Use inside CTOX:

- Read the review ask from `Latest user turn` and `Focus`.
- Use `Verified evidence`, `Anchors`, and the inspected diff as the basis for findings.
- This phase reports issues; it does not change CTOX policy or durable state rules.

- Inspect before judging.
- Focus on correctness, safety, security, data loss, deploy risk, performance, and missing tests.
- Only report issues you can support with evidence.
- Explain why each issue matters and how to verify it.
- Do not edit code in review phase unless the task changes.
