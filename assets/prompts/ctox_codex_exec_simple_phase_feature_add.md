Main meta-skill: `feature-add`

Use inside CTOX:

- Read the requested new behavior and finish rule from `Focus`.
- Use `Verified evidence` and `Anchors` to keep the feature inside the asked scope.
- If open follow-up work remains, it still belongs in CTOX runtime state.

- Name one requested behavior at a time.
- Find the smallest entry point.
- Add the minimum code for that behavior.
- Use one focused test or command to verify it.
- Then run the nearest related checks.
- Do not add extra options or speculative generalization.
